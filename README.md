1.왜 만들었는가? - 기존 GCS의 한계를 극복하기 위해 개발됨
<br>

국내 드론 산업은 다양한 분야에서 빠르게 성장하고 있다. 그러나 기존 GCS(지상관제시스템)은 다음과 같은 문제를 가지고 있다.

-통신거리의 한계로 원격 조종이 어려움.

-전문 GCS는 복잡하고 고가라 접근성이 낮음.

이 문제를 해결하기 위해, 클라우드 기반 저가형 드론 원격 제어 GCS 서버를 설계, 구현하였다.

2.무엇을 구현했는가? - 클라우드 기반 드론 제어 서버
<br>

두 논문에서 제안된 구조를 기반으로, 다음 요소들을 구현했다.

@전체 아키텍처 설계
-사용자 인터페이스 -> 서버 -> 드론 
(단, 실제 프로젝트에선 비용 문제로 드론 내에 셀룰러 네트워크를 탑재하지 못함. 대신 중계기를 이용해 서버 명령을 중계함)
-블루투스 기반 통신 중계(저가형 드론 대응. 서버->중계기->드론)
-서버는 명령 패킹,로그 저장, 중계기 통신을 담당

@서버 구현
-사용자 명령 수신 후 드론 프로토콜에 맞게 재패킹
-통신 내용을 기록하기 위한 로그 설계
-패킷 단위 명령 처리 및 오류 검출 로직 구현

@소스코드
-server : 사용자의 명령을 중계기로 전달하는 서버.
-receiver, client : 서버를 테스트하기 위한 테스트 프로그램.

3.어떤 점이 차별점인가?

@서버 중심 제어 구조로 확장성 확보
각각의 PC가 서버가 되는 엣지 컴퓨팅 방식은 불안정성이 컸지만, 클라우드 중심으로 통합 제어하는 구조를 통해 
장거리 제어가 가능하도록 설계했다.

@로깅 시스템
드론의 모든 이벤트를 시간, 발신지, 상태 메시지 단위로 저장하고 후속 자율비행까지 고려한 형태의 데이터 수집 구조를 설계하였다.

@실험 기반 안정성 검증
실험 결과, 명령 손실은 약 2.38% 였으며 드론 제어에 치명적인 영향은 없었다.

4.상세 구현

@핵심 기능
-사용자 인터페이스로부터 온 명령을 드론(중계기)로 전달
-수신 데이터를 로그 형태로 기록
-보안 기능

@사용 기술
-사용 라이브러리 : Boost asio
-개발 언어 : C++

@로직

-명령어 중계 :
사용자와 드론(중계기)은 서버에 접속할 때 각각 ID와 역할 정보를 전송한다. 서버는 이 ID를 기준으로 명령을 특정 대상에게 정확히 중계한다. 
사용자가 전송한 명령은 해당 ID의 큐에 적재되고, 드론(중계기)은 자신의 ID에 대응하는 큐에서 명령을 수신한다.
이러한 구조를 통해 사용자와 드론은 서버를 매개로 간접적으로 연결되므로, 서로의 접속 여부나 통신 상태에 직접 의존하지 않고 독립적으로 동작할 수 있다.

-로그 기록 :
서버에서 수행된 모든 통신 내역은 로그로 기록된다. 서버 관리 시 로그를 토대로 사용 기록, 오류 발생 여부를 명령어를 통해 실시간으로 확인할 수 있다.

-보안 기능 :
서버 접속 시 등록 ID에 대해 검사를 수행한다. 사용자 ID는 중복을 허용하지 않으며, 드론(중계기)의 경우 다중 드론 제어를 염두하여 중복을 허용한다. 
또한 사용자와 드론 ID는 서로 연결된 형태로 관리된다.
서버는 다음과 같은 경우 연결을 거부한다.

    1.이미 등록된 ID가 있는 경우 연결을 거부한다.

    2.또 다른 사용자가 이미 등록된(다른 사용자가 연결한) 드론 ID에 연결을 시도하는 경우 연결을 거부한다.

    3.중복된 드론 ID로 등록할 때, 이미 연결된 사용자 ID가 아닌 다른 ID로 등록을 시도하는 경우도 연결을 거부한다.

드론과 사용자가 접속을 종료했을 때 ID 등록이 해제된다.

5.결과와 성과

@클라우드 기반 저가형 드론 원격 제어
원거리에서 드론을 제어하는 시스템을 구축하였으며, 이 과정에서 드론 제어에 치명적인 영향이 발생하지 않았다.

@로깅 시스템
실시간 이벤트, 메시지, 상태 정보가 모두 기록되는 로깅 시스템을 구축하였으며,
자율 비행 및 반복 패턴 비행 연구에 활용 가능한 데이터를 수집할 수 있다.

@응용 확장성 증명
메쉬 네트워크 기반 다중 드론 제어 가능성을 제시하였으며, 산업 환경(농업, 측량, 재난 감시 등)에서 적용 가능성을 확인하였다.

@논문 작성 및 수상
연구 설계, 구현, 실험 분석이 모두 포함된 완전한 실증 프로젝트로 구성되었다.
결과를 바탕으로 논문을 작성하여 2024년 한국항행학회 종합학술대회에 출품하였으며, 
'클라우드 서버 기반 GCS를 통한 UAV 원격 제어' 논문이 우수논문상을 수상하였다.
